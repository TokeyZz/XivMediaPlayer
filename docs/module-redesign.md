# XivMediaPlayer 模块功能重设计

> **核心原则**: 先不修漏洞，重新定义每个模块的功能边界。记录所有改动，服务端用同样思路设计。

---

## 模块总览

| # | 模块 | 当前状态 | 目标 |
|---|------|---------|------|
| 1 | 媒体核心 | VLC 直接管理，错误不可见 | 播放生命周期清晰、错误可追踪 |
| 2 | 插件核心 | 同步逻辑散落、竞态多发 | 单一数据流向、状态机明确 |
| 3 | 服务端 | 服务端做播放控制 | 客户端权威、服务端只做转发 |
| 4 | 网络层 | 裸 HTTP、DTO 重复 | 统一通信协议 |
| 5 | URL 解析 | yt-dlp 裸调用 | 异步解析管道 |
| 6 | UI 窗口 | 窗口逻辑分散 | 事件驱动的 UI 层 |
| 7 | 渲染 & 配置 | 耦合在 Plugin 中 | 独立渲染管线 |

---

## 服务端设计指导原则（客户端改完后同步套用）

1. 服务端不主动做任何播放决策（不自动切歌、不推断进度）
2. 所有状态变更由客户端发起，服务端只做验证+转发
3. 需要新的 API 端点时，先定义接口契约 → 再实现 → 再对接
