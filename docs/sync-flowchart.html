7. 核心改动 vs 旧方案

场景旧方案 (Bug)新方案
seek 拖拽 任何人 seek 直接变成 DJ 并推送 seek 不推送、不夺权，心跳自然携带最新进度
DJ 离开 状态残留，DataAgeMs 无限增长，新客户端被强制同步过期状态 10s 心跳超时 → djDisconnected → 客户端忽略
自动切歌 服务端用 DataAgeMs + TimecodeMs 推算，暂停时也会误触发 服务端不再切歌，DJ 客户端监听 VLC Ended 主动切
两个 DJ 第二个静默覆盖第一个，无冲突检测 stateVersion CAS 校验，409 拒绝冲突请求
非 DJ 覆盖 任何客户端 POST /media 直接修改 /heartbeat 校验 ownerId + stateVersion
同 URL 重播 URL 不变 → 只 sync 分支 → 无法重启 VLC stateVersion 变化触发重播，不依赖 URL 比较
isLocalEnded 循环 VLC 失败 → Ended → isDifferentUrl → 重播 → 再失败 isDifferentUrl 只看 stateVersion+URL，加入重试上限
进度偏差 DataAgeMs 推算，时钟漂移 + 暂停导致巨大偏差 心跳携带真实 VLC timecode，无推算
新客户端进入 TimecodeMs+DataAgeMs 推算位置，yt-dlp 解析 30s 后已过时 心跳 2s 更新，新客户端拿到的时间码最多 2s 旧
